================================================================================
                         ПЛАН РАЗРАБОТКИ: SWIPE MATCH APP
                         Курсовой проект (Python + JS + PG)
================================================================================

ЭТАП 1: ФУНДАМЕНТ
Цель: Работающий прототип, где можно зарегистрироваться и сделать свайп.

[1] Настройка окружения 
     PostgreSQL: установка, создание БД, проверка подключения
     Backend: настройка FastAPI, SQLAlchemy, миграции
     Frontend: инициализация React, настройка роутинга
     Git: настройка веток, .gitignore

[2] База данных и Модели
     Таблица Users (id, username, age, gender, bio, photo_url)
     Таблица Swipes (swiper_id, swiped_id, is_like, created_at)
     Таблицы Blacklist / Whitelist
     Скрипт для заполнения БД тестовыми данными (фейковые юзеры)

[3] Бэкенд: API
     GET /candidates — выдача анкет (с фильтрацией)
     POST /swipe — сохранение лайка/дизлайка
     POST /auth — регистрация и вход (JWT или сессии)
     GET /profile — данные текущего пользователя

[4] Фронтенд: Основа 
     Страница входа / регистрации
     Главная страница со списком карточек
     Базовая верстка карточки (фото + имя)

[5] Интеграция 
     Подключение фронтенда к API
     Реализация кнопки "Лайк/Дизлайк" (без анимации пока)
     Проверка связки: БД <-> Бэк <-> Фронт
     ДЕДЛАЙН 1: Прототип готов к демонстрации основы 07.04

--------------------------------------------------------------------------------

ЭТАП 2: ФУНКЦИОНАЛ И ФИНАЛ 
Цель: Полноценное приложение с алгоритмами, дизайном и документацией.

[6] Алгоритмы и Логика 
     Реализация Черного списка (блокировка юзеров)
     Реализация Белого списка (приоритетный показ)
     Логика Мэтчей (если взаимный лайк → сохранение match)
     Исключение повторных показов (кто уже был в ленте)

[7] Фронтенд: Интерактив
     Анимация свайпов (библиотека react-tinder-card или аналог)
     Страница "Мэтчи" (список взаимных лайков)
     Настройки профиля (редактирование, загрузка фото)
     Управление списками (кнопки "В черный список")

[8] Тестирование и Отладка 
     Проверка краевых случаев (пустая база, кончившиеся анкеты)
     Исправление багов
     Оптимизация запросов к БД

[9] Финализация
     Скриншоты для README и пояснительной записки
     Финальный README (инструкция по запуску)
     Подготовка презентации / доклада
     ФИНАЛЬНЫЙ ДЕДЛАЙН, ПРОБНАЯ СДАЧА ПРОЕКТА 21.04
